# UIデザインシステム（スタイルガイド）

関連: [画面設計書](./03_screen-design.md) / [設計書](./02_design.md)

本アプリのUIは、角丸の柔らかいSaaS風UIを避け、**未来的なコントロールルーム（管制コンソール）風**の硬質で計器的なHUDテーマで統一する。オペレーターが大型コンソールでデータを監視・分析するような、情報密度の高い計器インターフェースを目指す。

> 注意: 本テーマは**視覚スタイル（黒背景・グリッド・斜めカット・六角形・蛍光グリーン・HUDステータス）**を指す。機能の呼称（モデル / 評価テーマ / 実行 / 採点 など）には軍事的なメタファー（「作戦ユニット」「ミッション」「戦況評価」等）を用いず、機能をそのまま表す素直な用語を使う。英字ラベルも `RUN` `MODELS` `DASHBOARD` `SCOREBOARD` のような中立的なテック用語に限定する（`MISSION` `DEPLOY` `UNIT` 等は使わない）。

### 用語マッピング（画面表記の基準）

| アプリ概念 | 日本語表記 | 英字ラベル例 |
| ---------- | ---------- | ------------ |
| LLMモデル | モデル | `MODELS` |
| 評価テーマ | 評価テーマ | `CASES` |
| ベンチマーク実行 | 実行 | `RUN` `RUNS` |
| Judge採点 | 採点 / 採点結果 | `JUDGE` `SCORE` |
| ダッシュボード | ダッシュボード | `DASHBOARD` |
| 実行履歴 | 実行履歴 | `RUN LOG` |
| エラー | エラー | `WARNING` `ERROR` |

画面上部に常時稼働する**コマンドバー（システムステータス・接続中プロバイダー数）**を置き、各パネルを計器のように並べる。

## 1. デザインコンセプト

| キーワード | 方針 |
| ---------- | ---- |
| 黒背景 | ベースは漆黒に近いダーク。情報を発光させる土台 |
| 濃いグレーのパネル | 情報のまとまりは濃灰のパネルで区切る |
| 蛍光グリーンのアクセント | 主要アクション・正常系・強調は蛍光グリーンで発光させる |
| オレンジ・赤の警告色 | 警告=オレンジ、危険/失敗=赤。明確に区別 |
| 斜めカットのパネル | パネル角はクリップパス等で斜めにカット（角丸禁止） |
| 六角形モチーフ | アイコン枠・バッジ・装飾に六角形を使用 |
| グリッド背景 | 背景に薄いグリッド線（方眼）を敷く |
| HUD風の情報表示 | 計器/管制画面のように数値・ラベルを配置 |
| ステータス表示 | `WARNING` / `SYSTEM ONLINE` / `ANALYZING` 等の英字ステータス |

### 禁止事項

- 角丸（`border-radius`）を持つ柔らかいカード・ボタン
- パステル調・低コントラストの淡い配色
- ドロップシャドウによるふんわりした浮遊感（代わりに発光/グローを使う）

## 2. カラーパレット

CSS変数（`globals.css` の `:root`）として定義する想定。

```css
:root {
  /* 背景 */
  --bg-base:        #05070A;  /* 漆黒ベース */
  --bg-panel:       #12161C;  /* 濃いグレーのパネル */
  --bg-panel-alt:   #1A1F27;  /* パネル内の段差 */
  --grid-line:      #1E2630;  /* グリッド線 */

  /* アクセント（蛍光グリーン） */
  --accent:         #39FF14;  /* 蛍光グリーン（主要強調・正常） */
  --accent-dim:     #1F8A0E;  /* 沈めたグリーン */
  --accent-glow:    rgba(57,255,20,0.35); /* グロー */

  /* 警告色 */
  --warn:           #FF8A00;  /* オレンジ（警告 / WARNING） */
  --danger:         #FF3B30;  /* 赤（危険 / 失敗 / ERROR） */

  /* 情報・テキスト */
  --text-primary:   #E6F0E8;  /* 主テキスト（ややグリーン寄りの白） */
  --text-muted:     #7C8A93;  /* 補助テキスト */
  --info:           #2FB8FF;  /* 実行中など中立情報（控えめに使用） */
  --border:         #2A333D;  /* パネル境界線 */
}
```

### 用途マッピング

| 用途 | 色 |
| ---- | -- |
| 主要ボタン / 正常 / SYSTEM ONLINE | `--accent`（蛍光グリーン） |
| 分析中 / ANALYZING | `--accent` 点滅 or `--info` |
| 警告 / WARNING / レート制限 | `--warn`（オレンジ） |
| 失敗 / ERROR / 危険操作（削除） | `--danger`（赤） |
| パネル背景 | `--bg-panel` |
| 全体背景 | `--bg-base` + グリッド |

## 3. タイポグラフィ

| 用途 | 指定 |
| ---- | ---- |
| ステータス・ラベル・数値 | 等幅フォント（`ui-monospace`, `JetBrains Mono`, `Share Tech Mono` 等）。`letter-spacing` を広め・**大文字** |
| 見出し | コンデンス系サンセリフ or 等幅。大文字＋トラッキング広め |
| 本文（回答全文等） | 可読性重視のサンセリフ。等幅は強制しない |

- ステータスラベルは常に大文字英字（例: `STATUS: ANALYZING`）。
- 数値（スコア・コスト・時間）はHUDらしく等幅で桁を揃える。

## 4. レイアウト要素

### 4.1 グリッド背景

全画面の最背面に薄いグリッドを敷く。

```css
.grid-bg {
  background-color: var(--bg-base);
  background-image:
    linear-gradient(var(--grid-line) 1px, transparent 1px),
    linear-gradient(90deg, var(--grid-line) 1px, transparent 1px);
  background-size: 32px 32px;
}
```

- 任意で中心からの放射グラデーションやスキャンライン（横走査線）を重ねてHUD感を強める。

### 4.2 斜めカットパネル

角丸の代わりに `clip-path` で角を斜めにカットする。

```css
.hud-panel {
  background: var(--bg-panel);
  border: 1px solid var(--border);
  /* 左上と右下を斜めカット */
  clip-path: polygon(
    14px 0, 100% 0, 100% calc(100% - 14px),
    calc(100% - 14px) 100%, 0 100%, 0 14px
  );
}
```

- 角丸（`border-radius`）は使用しない。
- パネル境界に蛍光グリーンの細いラインやコーナーマーカー（隅の `L` 字）を添えてHUD感を出す。

### 4.3 六角形モチーフ

アイコン枠・バッジ・装飾に六角形を使う。

```css
.hexagon {
  clip-path: polygon(25% 0, 75% 0, 100% 50%, 75% 100%, 25% 100%, 0 50%);
}
```

- モデルアイコン枠、ステータスインジケータ、ロゴ、空状態のプレースホルダ等に適用。
- 装飾として小さな六角形を散りばめた背景パターンも可（過剰にならない範囲で）。

### 4.4 HUDフレーム / コーナーマーカー

主要パネルの四隅に `⌐ ¬ L ⌐` 状のコーナーマーカーを配置し、計器枠を演出する。境界は1pxの硬いライン＋発光。

## 5. ステータス表示

ステータスは英字大文字のHUDラベルで表示する。色とアニメーションで状態を伝える。

| ステータス | 表示 | 色 | 演出 |
| ---------- | ---- | -- | ---- |
| 起動/正常 | `SYSTEM ONLINE` | 蛍光グリーン | 点灯（常時グロー） |
| 実行中 | `ANALYZING...` / `RUNNING` | 蛍光グリーン or 青 | 点滅 or プログレスバー走査 |
| 採点中 | `JUDGING...` | 蛍光グリーン | 点滅 |
| 警告 | `WARNING` | オレンジ | ゆっくり点滅 |
| 失敗/危険 | `ERROR` / `FAILED` | 赤 | 強い点滅 or 固定 |
| 待機 | `STANDBY` / `IDLE` | グレー | 消灯（dim） |

### 状態 → 実行ステータスの対応

| アプリ状態（[DB] status） | HUD表示 |
| ------------------------- | ------- |
| running | `ANALYZING...` |
| done（全成功） | `SYSTEM ONLINE` / `COMPLETE` |
| done（一部失敗） | `WARNING` |
| error | `ERROR` |

- APIキー未設定バナーは `WARNING: API KEY MISSING` をオレンジで表示。
- ステータスドット（六角形 or 円）＋ラベルのセットを基本形とする。

## 6. コンポーネント別ガイド

| コンポーネント | スタイル方針 |
| -------------- | ------------ |
| サイドナビ | 黒背景・等幅ラベル大文字。選択中は左に蛍光グリーンのバー＋発光 |
| パネル/カード | `.hud-panel`（斜めカット）。角丸禁止。隅にコーナーマーカー |
| ボタン（主要） | 蛍光グリーンの枠線＋黒地、ホバーで内側発光。斜めカット |
| ボタン（危険：削除） | 赤枠＋赤文字。ホバーで赤発光 |
| テーブル（スコア比較） | 罫線は細いグリッド線。数値は等幅右寄せ。1位行を蛍光グリーンでハイライト |
| バッジ（カテゴリ/ステータス） | 六角形 or 斜めカットタグ。英字大文字 |
| チャート（Recharts） | 背景透過・グリッド線は `--grid-line`、系列色は蛍光グリーン/オレンジ/青。グロー付き |
| 入力フォーム | 黒地＋細い枠線、フォーカスで蛍光グリーンの枠＋発光。角丸なし |
| ローディング | スピナーではなく走査バー or `ANALYZING...` の点滅 |
| 空状態 | 六角形プレースホルダ＋`NO DATA` ラベル |

## 7. アニメーション / 演出

- **グロー**: アクセント要素に `text-shadow` / `box-shadow` で発光。
- **点滅（ブリンク）**: 警告・分析中ステータスに低速ブリンク。
- **スキャンライン**: 実行中パネルに上下に走る走査線（控えめに）。
- **タイプライター**: ステータスログ（実行ログ）は1行ずつ流れる表示も可。
- 過度なモーションは避け、情報視認性を最優先する。

```css
@keyframes blink { 50% { opacity: 0.35; } }
.status-warning { color: var(--warn); animation: blink 1.2s step-start infinite; }
.glow { text-shadow: 0 0 8px var(--accent-glow); }
```

## 8. 実装メモ（Tailwind想定）

- Tailwind の `theme.extend.colors` に上記パレットを登録（`accent`, `warn`, `danger` 等）。
- `borderRadius` は既定の角丸を使わず、`rounded-none` を基本とする。
- 斜めカット・六角形は `clip-path` ユーティリティ（カスタムクラス or `plugin`）で実装。
- グリッド背景・グロー・コーナーマーカーは共通コンポーネント `HudPanel` / `GridBackground` / `StatusBadge` に切り出す。

### 追加想定の共通コンポーネント

| コンポーネント | 役割 |
| -------------- | ---- |
| `GridBackground` | グリッド＋スキャンラインの背景 |
| `HudPanel` | 斜めカット＋コーナーマーカー付きパネル |
| `StatusBadge` | 六角形ドット＋英字ステータスラベル |
| `HexIcon` | 六角形アイコン枠 |
| `NeonButton` | 蛍光グリーン/赤の発光ボタン |
