# セキュリティ設計書

関連: [要件定義書](./01_requirements.md) / [設計書](./02_design.md)

本アプリはローカル完結型のため、ネットワーク経由の攻撃面は限定的だが、**APIキー**と**プロンプトに含まれうる機密情報**の保護を重視する。

---

## 1. APIキー管理

APIキーは `.env.local` に保存する。

```env
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
GOOGLE_API_KEY=
```

### 1.1 禁止事項

| 禁止 | 理由 |
| ---- | ---- |
| DB保存 | DBファイル流出時の漏洩防止 |
| Git管理 | リポジトリ経由の漏洩防止（`.gitignore` で除外） |
| 画面上での平文表示 | ショルダーハック・スクショ流出防止 |
| ログ出力 | ログファイル経由の漏洩防止 |

### 1.2 取り扱い方針

- キーはサーバーサイド（API Routes）でのみ参照し、クライアントへ渡さない。
- 設定画面では**「設定済み / 未設定」のみ**を表示する（値は返さない）。
- 未設定プロバイダーのモデルを実行しようとした場合、実行前に `API_KEY_MISSING` で弾く（[API設計書 §6](./05_api-design.md#6-エラーコード一覧)）。
- エラーメッセージ・ログにキーを含めない。外部APIのエラーレスポンスを保存する際はキーを含む可能性のあるヘッダ等を除去する。

---

## 2. ローカルデータ保護

SQLite DBはローカルPC内に保存する。

```text
./data/benchmark.db
```

- `data/` は `.gitignore` で除外し、Git管理しない。
- バックアップはDBファイルの**手動エクスポート（コピー）**とする。
- DBには回答・スコアなどの評価データのみを保存し、APIキーは保存しない。

---

## 3. プロンプト内機密情報

評価テーマには実案件情報を入れる可能性があるため、登録前のマスキングを推奨する。

| 対象 | 対応 |
| ---- | ---- |
| 顧客名 | 匿名化 |
| IPアドレス | マスク |
| メールアドレス | マスク |
| 契約情報 | 削除 |
| 個人情報 | 削除 |

### 3.1 留意点

- プロンプトは外部LLM APIへ送信される。送信＝外部事業者へのデータ提供であることを利用者に明示する。
- テーマ登録画面に「機密情報はマスキングしてください」の注意書きを表示する。

---

## 4. 通信・実行制御に関する配慮

- 外部APIへの通信はHTTPS。SDK既定の検証を無効化しない。
- 同時実行数を最大3に制限し、コスト暴発・レート制限を抑制する（[設計書 §5](./02_design.md#5-実行制御)）。
- `AbortController` でタイムアウトを設け、ハングによるリソース占有を防ぐ。

---

## 5. 想定脅威と対策（簡易）

| 脅威 | 対策 |
| ---- | ---- |
| APIキー漏洩 | `.env.local` 管理・Git除外・非表示・非ログ |
| DBファイル流出 | キーを保存しない・機密はマスキング前提 |
| 機密情報の外部送信 | 登録前マスキングの推奨と注意喚起 |
| コスト暴発 | 同時実行制限・タイムアウト |
| 誤った大量実行 | 実行前の対象モデル・件数確認UI |

---

## 6. スコープ外（MVPでは非対象）

- ユーザー認証・認可（単一ユーザーのローカル利用前提）
- DB暗号化（必要なら将来 SQLCipher 等を検討）
- 監査ログ・アクセス制御
