## 3. 共通レイアウト

UIテーマは**未来的なコントロールルーム（管制コンソール）風**のHUD UIで統一する。詳細は[UIデザインシステム](./09_ui-design-system.md)を参照。全画面に共通の**コマンドバー（最上部）**と**サイドナビ**を備える。

```text
┌──────────────────────────────────────────────────────────┐
│ ◢ LLM BENCHMARK ARENA  ● SYSTEM ONLINE  PROVIDERS:3/4  ⚙ │ ← コマンドバー(HUD)
├──────────┬───────────────────────────────────────────────┤
│ ▸ NAV    │  (グリッド背景 + スキャンライン)                │
│ ──────── │                                               │
│ DASHBOARD│          メインコンテンツ（HUDパネル群）         │
│ CASES    │          ※斜めカットパネル・六角形モチーフ       │
│ MODELS   │                                               │
│ RUNS     │                                               │
│ SETTINGS │                                               │
└──────────┴───────────────────────────────────────────────┘
```

- **コマンドバー**: 最上部に常時表示。ロゴ＋システムステータス（`SYSTEM ONLINE`）＋接続中プロバイダー数（例 `PROVIDERS:3/4`）＋設定。司令室の管制ヘッダー。
- **サイドナビ**: 全画面共通。ラベルは英字大文字。現在ページは左に蛍光グリーンのバー＋発光でハイライト。
- 背景は全画面でグリッド＋（実行中パネルは）スキャンライン。
- APIキー未設定時はコマンドバー直下に `WARNING: API KEY MISSING`（オレンジ・明滅）の警告バナーを表示する。

## 4. 画面別仕様

### 4.1 ダッシュボード `/dashboard`

**目的**: モデル別の総合評価を俯瞰する。

レイアウト:

```text
┌── モデル別スコア ───────────────────────────┐
│ モデル        総合点  実行時間  推定コスト    │
│ Claude Opus    92     32秒     $0.12         │
│ GPT            89     28秒     $0.10         │
│ Gemini         84     25秒     $0.08         │
└──────────────────────────────────────────┘
┌── カテゴリ別ランキング ──┐ ┌── スコア推移グラフ ──┐
│ 開発     GPT            │ │  (Recharts 棒/折線)  │
│ 調査     Claude         │ │                     │
│ 設計     Claude         │ │                     │
│ コード生成 Codex         │ │                     │
└────────────────────────┘ └─────────────────────┘
```

- モデル別スコアは全実行の平均または最新を集計して表示。
- カテゴリ別ランキングはカテゴリごとの1位モデルを表示。
- グラフは Recharts で可視化。

主なAPI: `GET /api/runs`（集計はフロント側 or 集計API）。

---

### 4.2 テーマ一覧 `/cases`

レイアウト:

```text
┌────────────────────────────────────────┐
│ 評価テーマ           [+ 新規登録]        │
├────────────────────────────────────────┤
│ タイトル      カテゴリ   更新日   操作    │
│ Terraform構築  開発    06/01  [詳細][削除]│
│ 障害解析       障害解析  05/28  [詳細][削除]│
└────────────────────────────────────────┘
```

- カテゴリでのフィルタ・タイトル検索を備える。
- 「新規登録」で `/cases/new` へ。「詳細」で `/cases/[id]` へ。

主なAPI: `GET /api/cases`, `DELETE /api/cases/:id`。

---

### 4.3 テーマ登録 `/cases/new` / 編集

入力フォーム（コンポーネント: `CaseForm.tsx`）:

| 項目 | 入力UI | 必須 |
| ---- | ------ | :--: |
| タイトル | テキスト | ○ |
| カテゴリ | セレクト（開発/調査/設計/障害解析/ドキュメント作成/コードレビュー/セキュリティ調査） | ○ |
| プロンプト本文 | テキストエリア | ○ |
| 評価観点（Rubric） | テキストエリア（JSON可） | - |
| 想定する成果物 | テキストエリア | - |
| 備考 | テキストエリア | - |

- 入力検証: タイトル・カテゴリ・プロンプト本文は必須。
- 保存後はテーマ詳細へ遷移。

主なAPI: `POST /api/cases`, `PUT /api/cases/:id`。

---

### 4.4 テーマ詳細 `/cases/[id]`

```text
┌── テーマ詳細 ──────────────────────────┐
│ タイトル / カテゴリ / 更新日   [編集][削除]│
│ プロンプト本文（表示）                    │
│ 評価観点 / 想定成果物 / 備考              │
├── このテーマで実行 ─────────────────────┤
│ [対象モデル選択]  □GPT □Claude □Gemini  │
│ [Judgeモデル選択]  ▼ Claude Opus        │
│ 繰り返し回数: [1]      [▶ ベンチマーク実行]│
└──────────────────────────────────────┘
```

- 対象モデルは有効モデルから複数選択。
- Judgeモデルを1つ選択。
- 実行ボタン押下で `POST /api/runs` → 実行詳細 `/runs/[id]` へ遷移。

主なAPI: `GET /api/cases/:id`, `GET /api/models`, `POST /api/runs`。

```text
┌────────────────────────────────────────────────────┐
│ モデル管理                       [+ モデル登録]       │
```

入力フォーム（コンポーネント: `ModelForm.tsx`）:

| 項目 | 入力UI | 必須 |

### 4.6 実行履歴 `/runs`

実行ごとに**評価テーマの詳細**（カテゴリ・プロンプト抜粋）を表示するカード一覧。

```text
┌── RUN LOG ─────────────────────────────────────────┐
│ ┌────────────────────────────────────────────────┐ │
│ │ RUN-0002  [設計]                  ● COMPLETE   │ │
│ │ REST API設計（ECサイト）           MODELS: 2    │ │
│ │ ECサイトの商品・注文・在庫を扱う…    06/09 12:00 │ │
│ │ （プロンプト抜粋・2行で省略）        DETAIL →    │ │
│ └────────────────────────────────────────────────┘ │
│ ┌────────────────────────────────────────────────┐ │
│ │ RUN-0001  [障害解析]              ⚠ WARNING    │ │
│ │ メモリリーク調査 …                              │ │
│ └────────────────────────────────────────────────┘ │
└────────────────────────────────────────────────────┘
```

- 各カードに: 実行番号、カテゴリバッジ、テーマタイトル、プロンプト抜粋（2行省略）、ステータス、モデル数（失敗は `-N`）、実行日時。
- ステータスは `ANALYZING` / `COMPLETE` / `WARNING`（一部失敗）/ `ERROR` をHUDバッジで表示。
- カード全体がクリック可能で `/runs/[id]` へ遷移。

主なAPI: `GET /api/runs`（一覧表示は Server Component で Prisma から直接取得）。

### 4.7 実行詳細 `/runs/[id]`

**目的**: モデル間の回答・採点結果を比較し、**採点根拠を明示**する中核画面。

```text
┌── RUN SUMMARY ──────────────────────────────┐
│ ● COMPLETE  CATEGORY:設計  JUDGE:claude-..   │
├── EVALUATION TASK // 評価テーマ ─────────────┤
│ プロンプト全文 / 想定成果物 / 評価観点        │
├── SCORING RUBRIC // 採点基準（100点満点）────┤
│ 正確性/40 網羅性/20 実用性/20 可読/10 根拠/10 │
│ （各観点の意味を併記）                        │
├── SCORE MATRIX ──────────┬── SCORE CHART ────┤
│ モデル 総合/100 正確/40 …│  (棒グラフ)        │
│ Sonnet  85  37 14 17 10 7│                   │
│ Haiku   67  32  9 13  8 5│                   │
├── JUDGE RATIONALE // 採点根拠 ───────────────┤
│ ┌ Claude Sonnet 4.6           85 / 100 ────┐ │
│ │ 正確性 37/40 ▓▓▓▓  網羅 14/20 ▓▓ …       │ │
│ │ JUDGE COMMENT: 「…採点理由…」              │ │
│ │ ▶ 回答を確認する（クリックで回答全文を展開）│ │
│ └──────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

- **RUN SUMMARY**: ステータス・カテゴリ・Judgeモデル・モデル数。
- **EVALUATION TASK**: 送信したプロンプト全文・想定成果物・評価観点を表示。
- **SCORING RUBRIC**: 5観点の配点（合計100点）と各観点の意味を明示。
- **SCORE MATRIX**: 配点付きの項目別スコア表（総合/100・正確/40 …）。1位行をハイライト。
- **SCORE CHART**: モデル別総合点の棒グラフ（Recharts）。
- **JUDGE RATIONALE（採点根拠）**: モデルごとに、項目別スコアのバー＋Judgeコメント（採点理由）を常時表示。`▶ 回答を確認する` のアコーディオンで実行メトリクスと回答全文を開閉。
- 失敗モデルはエラー内容を表示。
- 「CSV EXPORT」で `GET /api/runs/:id/export` をダウンロード。

主なコンポーネント: `ScoreChart.tsx` / `ResponseAccordion.tsx`（回答全文アコーディオン）。

主なAPI: `GET /api/runs/:id`（画面は Server Component で Prisma から直接取得）, `GET /api/runs/:id/export`。

### 4.8 設定 `/settings`

```text
┌── APIキー状態 ────────────────────────┐
│ OpenAI     : ● 設定済み                │
│ Anthropic  : ● 設定済み                │
│ Google     : ○ 未設定（警告）          │
│ ※キーは .env.local で管理。画面では値を表示しない│
├── Judge設定 ──────────────────────────┤
│ 既定Judgeモデル: ▼ Claude Opus 4.8     │
│ 同時実行数: [3]                        │
└──────────────────────────────────────┘
```

- APIキーは**設定済み/未設定のみ**表示し、値は平文表示しない（[セキュリティ設計書](./07_security-design.md)参照）。
- 既定のJudgeモデル・同時実行数を設定。

## 5. UI共通仕様

テーマの詳細仕様（カラーパレット・斜めカット・六角形・グリッド背景・ステータス表示・コンポーネント）は[UIデザインシステム](./09_ui-design-system.md)に集約する。本セクションは概要のみ。

| 項目 | 仕様 |
| ---- | ---- |
| テーマ | 未来的なコントロールルーム（管制コンソール）風 HUD UI |
| 言語 | 画面文言は日本語、ステータス/ラベルは英字大文字（`SYSTEM ONLINE` 等） |
| カラー | 黒背景＋濃いグレーパネル。アクセント=蛍光グリーン、警告=オレンジ、危険/失敗=赤、中立=青 |
| 形状 | 角丸禁止。斜めカットパネル（clip-path）＋六角形モチーフ＋コーナーマーカー |
| 背景 | グリッド（方眼）背景。実行中パネルはスキャンライン |
| ステータス | `SYSTEM ONLINE` / `ANALYZING...` / `WARNING` / `ERROR` 等のHUDラベルで状態表現 |
| ローディング | スピナーではなく走査バー or `ANALYZING...` の明滅 |
| 空状態 | 六角形プレースホルダ＋`NO DATA` ラベル |
| エラー表示 | `WARNING`/`ERROR` のHUDアラート（オレンジ/赤・明滅）で表示 |
| レスポンシブ | デスクトップ優先（個人PC・大型コンソール利用前提） |
